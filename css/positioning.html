The Box Model

Learning about positioning in CSS requires a discussion about the box model. Every HTML element is considered to be a "box." The browser will see boxes (or: rectangles) of content to be display, whether they're paragraphs, headings, images or videos: this, in short, is the box model. Some boxes are displayed as block-level elements, and some as inline elements. Regardless, the following properties will affect how each box appears within the browser:

The Display Declaration

The display property because that also has an affect on where an item appears within its container. Ther are times

  • Each box is placed on its own line. This tells us that the 'div' tag is a block-level element by default. Block level elements take up an entire line in the browser, even if the element's width is narrower than its container. All header tags: h1, h2, etc. and paragraph tags are block-level elements by default.
  • Inline elements continue on in the same line. Think of thie term as "in the same line." The <em> tag (used for "emphasis" in a sentence) is an inline-level element by default.
  • We can change our boxes from their default block-level display to inline by adding the following style: 
  div {
      display: inline-block;
  }

Positioning can be applied to to any HTML element, but we'll stick with generic patterned boxes (using 'div' tags) for these examples to keep things simple.

Absolute

The absolute position declaration is just as obstinate as it sounds. It's for when you absolutely, positively, have to have something stuck in one damn place. Absolutely-positioned elements are removed from the document flow, meaning they will be placed within the browser according to their positioning statements and ignoring things like their containing elements and other elements that might also occupy that space. This is how you can layer one object over another: with the absolute position.

Fixed

The fixed position is similar to absolute, but it takes it a step further. Not only is the position attached to a particular location, it is attached to the location within the browser window. This means that the item will not scroll off the screen. It is primarily used for footers and annoying internet ads.

Inherit

Position declarations are not generally passed down to child elements (unlike declarations like font-size and background-color). The inherit position was designed so that you could do this.
