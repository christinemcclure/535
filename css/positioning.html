Positioning Elements

Positioning is such an important part of CSS that it deserves its own section. Here it is.

Learning about positioning in CSS requires about the box model. Every HTML element is considered to be a "box." The browser will see boxes of content to be display, whether they're paragraphs, headings, images or videos: this, in short, is the box model. You'll get a longer tutorial in the Javascript section when you learn about the Document Object Model, but the basics are this regarding positioning:

  1. When the browser loads your web document, it reads from top to bottom, left to right.
  2. The elements are initially laid out on the page in that order. If you have an <H3> tag listed before an <H1> tag, the <H3> tag will be displayed first.
  3. Once the elements are laid out out in the browser, your CSS rules will be applied to the document. Here is were you can say that elements 1-2-3 should display as 3-1-2 if you like.
  4. This usually happens far too quickly to see that elements get repositioned on the screen.

The Display Declaration

The display property because that also has an affect on where an item appears within its container. Ther are times

  • Each box is placed on its own line. This tells us that the 'div' tag is a block-level element by default. Block level elements take up an entire line in the browser, even if the element's width is narrower than its container. All header tags: h1, h2, etc. and paragraph tags are block-level elements by default.
  • Inline elements continue on in the same line. Think of thie term as "in the same line." The <em> tag (used for "emphasis" in a sentence) is an inline-level element by default.
  • We can change our boxes from their default block-level display to inline by adding the following style: 
div {
    display: inline-block;
}
